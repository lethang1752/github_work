***Note: Đảm bảo các tham số giống CSDL cũ (Source): db_files, language code, process, ...

- Tạo phân vùng chứa tool M5 và giải nén (thực hiện trên cả source db và dest db)
oracle $: mkdir /u01/app/oracle/m5
oracle $: unzip /home/oracle/2999157.1-DBMIG.ZIP-DBMIG.zip -d /u01/app/oracle/m5

- Chỉnh sửa code của tool M5 theo file đính kèm trong folder (thực hiện trên cả source db và dest db)

- Thêm file dbmig_ts_list.txt chứa tên các tablespace sẽ chuyển (ngoại trừ SYSTEM, SYSAUX, TEMP, UNDOTBS) trong thư mục m5/cmd

- Thực hiện cấu hình passwordless giữa source db và dest db, thực hiện trên source db
oracle $: cd /home/oracle/.ssh
oracle $: ssh-keygen -t rsa
oracle $: ssh-copy-id oracle@dest-db

- Cấu hình tắt index trên ZDLRA, thực hiện trên máy chủ RA
oracle $: sqlplus rasys/welcome1
create or replace view RAI_STALL_WHEN
AS
SELECT task_id
FROM task
WHERE task_type = 140
AND db_key = <DB_KEY>;

EXEC DBMS_RA_INT.config(p_name=>'_stall_when', p_value=>'ON', comments=>'Enabled for Cross Endian Migration');

- Cấu hình wallet cho dest db
    + Trên source db, export wallet và kiểm tra key
    oracle $: sqlplus / as sysdba
    ADMINISTER KEY MANAGEMENT EXPORT ENCRYPTION KEYS WITH SECRET "welcome1" TO 'exportkeys.exp' FORCE KEYSTORE IDENTIFIED BY NPC$12345;
    SELECT KEY_ID FROM V$ENCRYPTION_KEYS;

    + Trên source db, scp export key sang dest db
    oracle $: scp exportkey.exp oracle@dest-db:/home/oracle/

    + Trên dest db, tạo wallet và thêm đường dẫn wallet vào sqlnet.ora để csdl nhận wallet mới
    oracle $: mkdir -p /u01/app/oracle/wallet
    oracle $: cd /u01/app/oracle/wallet
    oracle $: mkstore -wrl /u01/app/oracle/wallet -create

    + Trên dest db, thực hiện import key và check key 
    oracle $: sqlplus / as sysdba
    ADMINISTER KEY MANAGEMENT IMPORT KEYS WITH SECRET "welcome1" FROM 'exportkeys.exp' IDENTIFIED BY NPC$12345 WITH BACKUP;
    SELECT KEY_ID FROM V$ENCRYPTION_KEYS;

    + Trên dest db, thực hiện activate key 
    oracle $: sqlplus / as sysdba
    ADMINISTER KEY MANAGEMENT USE KEY <KEY_ID> USING TAG 'Activate Key on target' FORCE KEYSTORE IDENTIFIED BY NPC$12345 WITH BACKUP;
